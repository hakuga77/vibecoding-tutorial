# ハンズオン進行ガイド

> このファイルはファシリテーター向けの進行台本です。

---

## タイムライン（全体 1.5〜2時間）

| 時間 | 内容 | 目安 |
|------|------|------|
| 0:00〜0:15 | オープニング・VibeCoding の紹介 | 15分 |
| 0:15〜0:30 | 環境確認・Codex App 起動 | 15分 |
| 0:30〜0:45 | ライブデモ（ファシリテーターが実演） | 15分 |
| 0:45〜1:20 | ハンズオン（参加者が自由に改造） | 35分 |
| 1:20〜1:30 | 成果発表・共有 | 10分 |
| 1:30〜     | まとめ・Q&A | 〜30分 |

---

## パート1：オープニング（0:00〜0:15）

### ゴール
参加者が「VibeCoding とは何か」をざっくり理解する。

### 進め方
1. 自己紹介・勉強会の趣旨説明（5分）
2. `docs/01-what-is-vibecoding.md` の内容をベースに説明（10分）
   - 「今日はコードを一行も書きません」
   - 「AIに話しかけるだけでアプリが変わります」
   - 「業務に使えるイメージを持って帰ってください」

### ポイント
- 難しい技術用語は使わない
- 「魔法みたいなもの」「料理のレシピをAIに伝える感じ」などのたとえを使う

---

## パート2：環境確認（0:15〜0:30）

### ゴール
全員が Codex App を起動して、アプリをブラウザで開けている状態にする。

### 進め方
1. 「`starter-app/index.html` をブラウザで開いてください」
2. 「Codex App を起動して、フォルダを開いてください」
3. 困っている方のサポートをする

### よくある問題
- Git が入っていない → 当日サポート用に USB や共有フォルダでの配布も検討
- Codex App にログインできない → 事前にアカウント作成を依頼しておく

---

## パート3：ライブデモ（0:30〜0:45）

### ゴール
「こうやって使うんだ」というイメージを参加者全員に見せる。

### 実演の流れ（例）

**デモ1：デザインを変える（5分）**
```
Codex への指示：
「このアプリの背景色を薄いブルーにして、タイトルを大きく表示してください」
```
→ ファイルが更新されたことを見せる → ブラウザをリロードして変化を確認

**デモ2：機能を追加する（5分）**
```
Codex への指示：
「出発日から帰着日まで何泊するか自動で計算して表示してください」
```
→ 泊数が自動計算される様子を見せる

**デモ3：壊れても戻せる（5分）**
```
Codex への指示：
「全部ピンク色にして、文字もピンクにして」（わざと見づらくする）
「さっきの状態に戻してください」
```
→ 「間違えても大丈夫」というメッセージを伝える

### ファシリテーターへのアドバイス
- 意図的に失敗・修正を見せると親近感が湧く
- 「私もはじめはうまくいきませんでした」というエピソードがあると効果的

---

## パート4：ハンズオン（0:45〜1:20）

### ゴール
参加者それぞれが Codex を使って「出張プランナー」を自分好みに改造する。

### 進め方
1. 「では自由に改造してみましょう！」と声をかける
2. `docs/04-prompt-tips.md` を参照するよう案内する
3. 机間巡回してサポートする

### 改造のアイデア（参加者に提示する）
- 「持ち物リストをカテゴリ別（衣類・仕事道具・日用品）に分けて」
- 「出発日まで何日か残りをカウントダウン表示して」
- 「予算を入力できる欄と、1日あたりの費用を計算して表示して」
- 「印刷できるボタンを追加して」
- 「ダークモード（黒背景）にして」

### ファシリテーターのサポートポイント
- うまく指示が伝わらない場合：「もっと具体的に言い直してみましょう」
- Codex がエラーを返した場合：「そのまま Codex に"直して"と伝えてみて」
- 行き詰まっている場合：一緒にプロンプトを考える

---

## パート5：成果発表（1:20〜1:30）

### 進め方
1. 数名に自分の画面を共有・発表してもらう（2〜3分/人）
2. 「どんな指示を出したら変わりましたか？」と質問する
3. ユニークな改造は全員に見せる

### 盛り上げのポイント
- どんな小さな変化でも褒める
- 「それは面白い発想ですね！」と反応する

---

## まとめ・Q&A（1:30〜）

### まとめで伝えること
1. 「今日コードを書いた人はいますか？→ 誰もいない！でもアプリが変わりました」
2. VibeCoding は業務でも使える（例：社内向け簡易ツール・レポートテンプレート等）
3. 大事なのは「何を作りたいか」を明確に言語化すること
4. 公式ドキュメントや Q&A の案内

### よくある質問
**Q: セキュリティは大丈夫？**
A: 社外秘情報を Codex に貼り付けるのは避けてください。開発用途での利用は一般的に問題ありませんが、自社のルールを確認してください。

**Q: 作ったアプリは保存される？**
A: ローカルのファイルとして保存されます。Git でバージョン管理することも可能です。

**Q: 業務で本格的に使いたい場合は？**
A: チームや部署での活用方法を AI 活用推進チームに相談してください。

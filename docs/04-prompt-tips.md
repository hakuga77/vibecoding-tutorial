# Codex への上手な指示の出し方

---

## 良いプロンプトの3つの原則

### 1. 具体的に伝える

| NG | OK |
|----|-----|
| 「いい感じにして」 | 「背景を白にして、タイトルの文字を青色の太字にして」 |
| 「使いやすくして」 | 「入力フォームの幅を広げて、ボタンを画面中央に配置して」 |
| 「かっこよくして」 | 「シンプルでビジネスらしい落ち着いたデザインにして」 |

### 2. 一度に一つお願いする

複数のことを同時にお願いすると、意図した通りにならないことがあります。

| NG | OK |
|----|-----|
| 「デザインを直して機能も追加して印刷もできるようにして」 | まず「デザインを直して」→ 確認 → 次に「機能を追加して」 |

### 3. うまくいかなければ言い直す

Codex は何度でも修正してくれます。

- 「やっぱり元に戻して」
- 「もっとシンプルにして」
- 「さっきと同じ方法で別の部分も直して」

## 出張プランナー 拡張プロンプト集

### デザイン系

```
背景色を薄いグレーにして、カードのような見た目にしてください
```

```
タイトルをもっと大きく、太字で表示してください
```

```
ボタンの色を濃い青にして、マウスを乗せたときに少し暗くなるようにしてください
```

```
全体的にビジネス向けのシンプルで洗練されたデザインにしてください
```

```
スマートフォンでも見やすいレイアウトにしてください
```

### 機能追加系

```
出発日から帰着日まで何泊するか自動で計算して「〇泊〇日」と表示してください
```

```
出発日まで残り何日かをカウントダウンで表示してください
```

```
予算を入力できる欄を追加して、1日あたりの予算を自動計算して表示してください
```

```
「印刷する」ボタンを追加して、クリックするとブラウザの印刷画面が開くようにしてください
```

```
入力した内容を一覧でまとめた「旅程サマリー」を下部に表示してください
```

### 持ち物チェックリスト系

```
持ち物リストを「仕事道具」「衣類」「日用品」のカテゴリに分けて表示してください
```

```
チェックボックスをクリックしたとき、チェックした項目が薄い色に変わるようにしてください
```

```
「自分でメモを追加」できる入力欄をチェックリストの下に追加してください
```

```
「すべてチェック解除」ボタンを追加してください
```

## 困ったときのプロンプト

### エラーが出たとき
```
エラーが出て動かなくなりました。直してください
```

### 意図と違う結果になったとき
```
イメージと違いました。〇〇の部分だけ元に戻してください
```

### どうすればいいかわからないとき
```
〇〇を実現したいのですが、どのように指示すればよいですか？
```

---

## 発展編：もっと上手に使うために

### 「なぜ」を説明する

目的を伝えると、より適切な実装をしてくれます。

```
印刷して上司に提出したいので、印刷時にきれいに見えるレイアウトにしてください
```

### 制約を伝える

```
HTML・CSS・JavaScript だけで実装してください（サーバーは使いません）
```

### ステップに分けて依頼する

```
まずデザインだけ変えてください。動作は後で変えます
```
